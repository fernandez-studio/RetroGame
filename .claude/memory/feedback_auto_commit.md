---
name: Auto-commit and memory saves
description: Automatically commit and save memory after features work, without waiting for user
type: feedback
---
Automatically commit to git and update memory whenever a feature is working — don't wait for the user to ask or remind you.

**Why:** Token limits can cut sessions off without warning. Users don't naturally think in checkpoint terms while building. Losing work mid-session is a bad experience.

**How to apply:** After any feature is tested and working in the browser, commit it and update memory immediately, as part of completing the task — not as a separate step.
